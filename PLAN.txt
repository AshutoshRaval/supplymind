SupplyMind — 3-Day Action Plan
================================

PROJECT OVERVIEW
----------------
Build an inventory management system using a RAG pipeline with ReAct agents.

Core problem: When stock drops below threshold, the system:
  1. Alerts that a restock is needed
  2. Fetches quotes from available vendors
  3. Ranks and recommends the best option
  4. Drafts a Purchase Order

Tech stack:
  - LangGraph         (agent orchestration, supervisor graph)
  - LangChain Anthropic (LLM calls)
  - SQLAlchemy + SQLite (inventory database)
  - ChromaDB          (vector store for RAG)
  - MCP               (expose DB tools to external clients)
  - A2A               (agent-to-agent communication protocol)


ARCHITECTURE
------------

  User / CLI
      |
      v
  LangGraph Supervisor Graph
      |-- InventoryMonitor Agent   <-- written by developer
      |       |__ Tools: check_stock, get_consumption_history
      |
      |-- VendorAdvisor Agent      <-- written by developer
              |__ Tools: get_quotes, rank_vendors, recommend_order
                           |
                           v
                     MCP Server (exposes DB as tools to external clients)
                           |
                           v
                     A2A: VendorAdvisor <-> external PriceBot agent

RAG layer sits underneath — semantic search over item + supplier descriptions.


THE 2 AGENTS (written by the developer)
----------------------------------------

Agent 1 — InventoryMonitor  (Day 1, simpler)
  - Input   : none (pulls from DB itself)
  - Location: graph/inventory_agent.py
  - ReAct loop:
      1. call check_stock_levels
      2. if below threshold -> call get_consumption_velocity
      3. compute days-to-stockout
      4. emit structured alert
  - Key learning: how tools feed back into agent reasoning

Agent 2 — VendorAdvisor  (Day 2, more complex)
  - Input   : item_id + quantity_needed (from InventoryMonitor alert)
  - Location: graph/vendor_agent.py
  - ReAct loop:
      1. call get_supplier_quotes
      2. call rank_vendors
      3. optionally call RAG search for historical context
      4. output recommendation dict
  - Key learning: multi-tool chaining, structured output, agent handoff

CONCEPTS COVERED
----------------
  - ReAct Agents    : Reason + Act loop, tools feed back into agent reasoning
  - Agent Tools     : @tool decorated functions (check_stock, get_quotes, rank_vendors)
  - Skills          : Reusable prompt + tool compositions (reorder skill, RAG skill)
  - MCP             : Model Context Protocol server exposing DB tools to any client
  - A2A             : Agent-to-Agent protocol, agents call each other by name not import
  - LangGraph       : StateGraph with supervisor routing between multiple agents
  - RAG Pipeline    : ChromaDB vector store, semantic search over item/supplier descriptions
